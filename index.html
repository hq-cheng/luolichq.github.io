GMapping SLAM基础、原理与实现（四）：概率学基础知识、贝叶斯滤波器、以及粒子滤波器

4.1 应用概率学知识解决机器人SLAM问题

SLAM问题本质上是状态估计问题，其核心问题就是由SLAM数学模型对移动机器人位姿进行估计，同时完成环境地图的构建。第三节得到的SLAM数学模型为：

xt=f(xt−1,ut,wt)x_t = f(x_{t-1}, u_t, w_t) x​t​​=f(x​t−1​​,u​t​​,w​t​​)

zt=h(xt,mt,vt)z_t = h(x_t, m_t, v_t) z​t​​=h(x​t​​,m​t​​,v​t​​)

  • 状态值：xtx_tx​t​​；
  • 观测值：ztz_tz​t​​；
  • 控制输入值（里程计相邻时刻之间的变化值）：utu_tu​t​​；
  • 环境地图值（地图中的路标点，或特征点）：mtm_tm​t​​；
  • 噪声：wtw_tw​t​​，vtv_tv​t​​；

先暂时不考虑环境地图构建问题，将状态值xtx_tx​t​​和mtm_tm​t​​看作是同一个状态变量，主要对移动机器人的位姿估计问题进行分析，分析的方法是将整个SLAM数学模型中所有的物理量看作随机变量，引入概率学知识，对状态估计问题进行求解。因此，SLAM数学模型简化为：

xt=f(xt−1,ut,wt)x_t = f(x_{t-1}, u_t, w_t) x​t​​=f(x​t−1​​,u​t​​,w​t​​)

zt=h(xt,vt)z_t = h(x_t, v_t) z​t​​=h(x​t​​,v​t​​)

post @ 2019-10-17

3.1 里程计运动模型

由于机器人的位姿也可以在底盘坐标系下表示，所以有两种求解运动模型的思路。里程计（编码器）一般测量到的信息是底盘在每两个时间间隔内位置和方位的变化量：

  • 当我们获取的里程计信息是相对于世界坐标系的话，则运动模型输入为[dx,dy,dθ]wT[dx, dy, d\theta]_w^T[dx,dy,dθ]​w​T​​；
  • 当我们获取的里程计信息是相对于底盘基准坐标系的话，则运动模型输入为[dx,dy,dθ]robotT[dx, dy, d\theta]_{robot}^T[dx,dy,dθ]​robot​T​​

后者更为常用，因为我们在SLAM算法中一般是通过监听底盘基准坐标系与里程计坐标系之间的tf变换关系来获取里程计信息，所以得到的都是指定时刻里程计坐标下的机器人位姿（里程计和底盘之间就一个简单的静态坐标变换）。

GMapping SLAM基础、原理与实现（二）：激光雷达与数据获取

post @ 2019-10-17

2.1 什么是激光雷达

激光雷达（LiDAR, Light Detection And Ranging）是激光探测与测距的缩写。

激光雷达使用了脉冲激光照射目标，可以根据照射所反射回来的脉冲激光测量雷达到达目标的距离。在反射的脉冲激光中也会带有相关信息，激光雷达可以根据返回时间和波长的差异来对目标进行3D建模。另外，激光雷达一般用于SLAM技术中，用于制作高精度地图。激光雷达所提供的点云图是直接含有深度信息的，所以使用激光雷达进行场景建图会更加地容易实现。
